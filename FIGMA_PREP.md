# Как подготовить макет в Figma

Чеклист для дизайнера перед передачей макета на вёрстку.
Выполнение этих правил напрямую влияет на точность и чистоту результата.

---

## 1. Фрейм страницы

- **Один фрейм = одна страница** — не несколько кусков на холсте
- Ширина фрейма — **1440px** (desktop)
- Название фрейма — понятное: `Case — Название проекта` или `Page — Blog`
- Фрейм должен **полностью рендериться**: все шрифты подключены, компоненты не сломаны, изображения видны — скриншот фрейма используется для финальной сверки

---

## 2. Именование слоёв ← самое важное

Верстальщик читает слои, чтобы понять структуру страницы. Чем точнее названы слои, тем точнее результат.

**Секции** — верхний уровень фрейма:
```
Hero
Project Meta
Challenge
Process
Results
CTA
```

**Группы внутри секций** — по смыслу:
```
card-list / card-item
tag-list / tag
image-block / caption
```

**Избегать**: `Frame 123`, `Group 45`, `Rectangle`, `Ellipse 2` — это создаёт неопределённость в структуре.

---

## 3. Auto Layout на секциях

Если секции сделаны с **Auto Layout** (вертикальное направление, padding, gap) — верстальщик получает точные значения отступов как flex-свойства, а не угадывает их по координатам.

Применяй Auto Layout минимум на:
- Каждой секции страницы
- Карточках и повторяющихся элементах
- Контентных блоках с текстом

---

## 4. Тексты

- Все тексты — **настоящий текст**, не растрированные изображения
- Заголовки, параграфы, теги, кнопки, подписи, метаданные — всё редактируемо
- Текст не слит с фоновой картинкой в один слой

---

## 5. Цвета и фоны

- Уникальный фон секции — **fill на фрейме секции**, не отдельный прямоугольник поверх
- Акцентный цвет кейса (если есть) — хотя бы один элемент с ним, чтобы верстальщик его считал
- Стандартные белый/чёрный указывать не нужно — они есть в design tokens

---

## 6. Картинки

- Финальные изображения (скриншоты продукта, фото) — вставлять **реальными**, не заглушками
- Если изображений ещё нет — ставить прямоугольник с текстовой подписью внутри:
  ```
  [скриншот главного экрана — светлая тема]
  [фото рабочего процесса]
  ```
  Верстальщик вынесет их списком в итоговый отчёт
- Не использовать картинку там, где подразумевается текстовый контент

---

## 7. Адаптив

Если нужна точная адаптивная вёрстка — **отдельные фреймы** рядом на холсте:

| Фрейм | Ширина |
|---|---|
| Desktop | 1440px |
| Tablet | 991px |
| Mobile | 375px |

Названия: `Case — Название / Desktop`, `Case — Название / Tablet`, `Case — Название / Mobile`

Если адаптив стандартный и не требует особого внимания — достаточно одного desktop-фрейма.

---

## 8. Навигация / хедер кейса

- Хедер **светлый** (стандартный) → передать параметр `light` (по умолчанию)
- Хедер **тёмный** (белые иконки, чёрный фон) → передать параметр `dark`
- Сам nav в макете прорабатывать не нужно — он берётся из компонента

---

## 9. Ссылка на Figma

Давать ссылку на **конкретный фрейм**, не на весь файл.

Как получить:
1. Кликнуть на фрейм
2. Правая кнопка → *Copy link to selection*
   (или скопировать из адресной строки — там появится `node-id`)

Нужный формат:
```
https://www.figma.com/design/XXXXX/имя-файла?node-id=0-123
```

Если есть отдельные фреймы для tablet/mobile — давать ссылки на все три.

---

## 10. Что не нужно делать в макете

- Прорабатывать hover-состояния
- Размечать точные px-отступы — верстальщик ориентируется по Auto Layout и пропорциям
- Делать мобильный фрейм, если адаптив стандартный
- Прорабатывать nav и footer — они берутся из компонентов
