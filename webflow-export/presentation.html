Проблема
Что мешало нормально работать
  • Тысячи строк инлайн-стилей на каждой странице — скопированы из Webflow как есть
  • Нет общей таблицы стилей: любое изменение — вручную на 28 страницах
  • Навигационные дропдауны сломаны вне Webflow (Webflow JS не работал)
  • overflow-x: hidden блокировал position: sticky — навигация не залипала

  • Нет системы для добавления новых страниц из Figma
Решение 1 из 3
Единая таблица стилей

Один файл styles.css подключён ко всем 28 страницам. Содержит design tokens и систему классов с префиксом mtk-

:root { --mtk-color-accent: #FF7379; --mtk-color-sand: #F3EFED; --mtk-font-display: 'Outfit', sans-serif; --mtk-font-serif: 'Spectral', serif; --mtk-space-section: 64px; --mtk-max-width: 1440px; }
Ключевое открытие
overflow-x: hidden создаёт scroll container и ломает position: sticky.

Заменили на overflow-x: clip — поведение то же, но sticky работает корректно.

Решение 2 из 3
Система компонентов

7 HTML-файлов в папке components/ — повторно используемые блоки. Новая страница собирается как конструктор.

Исправления
4 поведения навигации
Peek nav
При скролле вниз навигация скрывается. При скролле вверх — плавно выезжает и залипает сверху. Работает на всех 28 страницах через styles.css + один JS-блок в _scripts.html
Sticky tags bar
На странице Services теги-якоря (UI Design, Branding...) залипают при скролле по секциям и отлипают в конце. Стало возможным после замены overflow-x: hidden → clip
Маркиз-карусель
На странице Design Subscription секция Selected Work — бесконечная карусель кейсов справа налево. Webflow-анимации заменены на CSS @keyframes + JS-дублирование карточек
Nav gap и spacing
Расстояние между пунктами меню: 40px на десктопе, 28px на планшете (было 16px). Отступ 72px между навигацией и контентом на About, Portfolio, Contacts
Контактная форма
Webflow Forms → Netlify Forms
Было
Форма отправляла данные на серверы Webflow. Вне Webflow-хостинга — ошибка при каждой отправке
Стало
Netlify Forms: data-netlify="true", уведомление на hello@motka.design
Нетривиальный момент
Webflow JS перехватывал событие submit раньше Netlify и отправлял данные себе.

Решение: собственный обработчик с capture: true + stopImmediatePropagation() — перехватывает submit первым и отправляет в Netlify через fetch
Решение 3 из 3
AI-скилы для верстки

Два slash-команды для Claude Code. Верстают новую страницу по Figma-макету без написания кода вручную.

/build-page
Root-страницы: About, Services, Blog и любые новые. Аргументы: Figma URL + имя файла
/build-case
Кейсы портфолио. Дополнительный параметр: light / dark хедер
Как работает
1. Читает Figma через MCP API
2. Считывает имена слоёв → понимает структуру секций
3. Делает скриншот для финальной сверки
4. Явно перечисляет секции до начала вёрстки
5. Читает компоненты, собирает HTML
6. Сравнивает результат со скриншотом
Figma → Code
Как подготовить макет

Чтобы вёрстка получилась точной — нужна правильная подготовка. Правила зафиксированы в FIGMA_PREP.md

Фрейм и слои
Один фрейм = одна страница, 1440px. Секции называть осмысленно: Hero, Challenge, Results — не Frame 123
Auto Layout
На каждой секции — Auto Layout с вертикальным направлением и gap. Верстальщик получает flex-значения вместо абсолютных координат
Адаптив и ссылка
Ссылка на конкретный фрейм с node-id. Если нужен точный адаптив — отдельные фреймы 1440 / 991 / 375px
