## 1. Цель этапа ①

Получить работающий Astro-проект с дизайн-системой, извлечённой из утверждённого макета, и **pixel-perfect** свёрстанной первой страницей — лендингом «Design Subscription».

**Источник истины по дизайну:** Figma
`https://www.figma.com/design/AKLK947vSjYe8BFz9wGO0Q/motka.design-06-26---dev?node-id=1-11872`
Исходный артборд: **1600 × 8000 px**, но контент уже лежит в едином контейнере **1440** (поля по 80px). База вёрстки — **1440** (см. §4а).

### Критерии успеха (verifiable)

1. `npm run dev` поднимает проект, `npm run build` собирает статику в `dist/` без ошибок.
2. Главная страница `/` рендерит все 12 секций лендинга.
3. **Pixel-perfect на десктопе 1440:** скриншот собранной страницы (viewport 1440) измерен кодом (PIL), ключевые величины (ширина контейнера, размеры/отступы секций, типографика) совпадают с Figma-копией 1440; все расхождения зафиксированы с px-дельтой и устранены (кроме суб-пиксельных различий рендера шрифтов).
4. **Адаптив** на брейкпойнтах 1280 / 1024 / 768 / 390 не ломается (нет горизонтального скролла, контент читаем, секции рефлоутся осмысленно). Поскольку мобильного макета нет — раскладки на моё усмотрение, ревью по результату; pixel-perfect к ним НЕ применяется.
5. Все цвета/шрифты/отступы берутся из `tokens.css`, а не хардкодятся в компонентах.

## 2. Технический стек

- **Astro** (статический вывод, ноль клиентского JS по умолчанию).
- Чистый CSS на CSS-переменных (без Tailwind/UI-китов — у нас свой дизайн).
- Шрифты (источники подтверждены):
  - **Factor A** (дисплейный) — самохостим, 2 начертания (Regular 400, Bold 700, `.woff`), берём с Motka Webflow CDN: `6511a3eee51f3537baa144f9_Factor A-Regular-Web.woff`, `6511a3eec2a43dc012f581c1_Factor A-Bold-Web.woff`. Кладём в `src/assets/fonts/`, объявляем `@font-face` локально (по возможности конвертируем в woff2).
  - **Spectral** (серив) — Google Fonts, самохостим (woff2).
- Изображения: оптимизация через Astro Assets (`astro:assets`).

## 4а. База 1440 + аудит качества сборки макета

### Копия на 1440 (первый шаг реализации)

Контент макета уже построен на сетке **1440** (артборд 1600 = контейнер 1440 + поля 80px; вся обёртка — один фрейм `1:11873` в x=80, внутренний гуттер 40px → контент 1360).

**Делаем копию на 1440** (требует Figma write → перед `use_figma` обязательно загрузить скил `figma-use`):
1. Дублировать корневой фрейм `1:11872`.
2. Ширина копии = **1440**.
3. Сдвинуть обёртку `1:11873` из `x=80` в `x=0` → контент edge-to-edge 1440 (контент 1360 + гуттер 40px по краям).
4. Визуально проверить (screenshot), что ничего не уехало; починить сдвиги, если есть.

Дальше **вся вёрстка опирается на копию 1440**, числа снимаются с неё (без мысленного −80).

### Выводы аудита (вердикт: сборка средняя)

**Хорошо:** 0 настоящих групп (всё на фреймах), 83 инстанса (кнопки, инпуты, статус-бары — компоненты), единая сетка-контейнер 1440/1360.

**Риски для вёрстки и адаптива (учитываем):**
1. **Авто-лейаут не гарантирован** — 155 фреймов с именами «Group …» (бывшие группы), глубина до 19. → Лейаут секций под адаптив **пересобираем руками** (flex/grid), не доверяем авто-лейауту по всему дереву.
2. **Hero — горизонтальная лента 4559px** (`card works`, x=429) → горизонтальный скролл/маска; на узких брейках логику показа переосмыслить.
3. **Разнобой гуттеров:** 40px почти везде, 80px в `process block` (`1:20339`), x=87 в `Heading` (`1:18213`) → привести к единому контейнеру.
4. **Дубли имён** (`Coverage block` ×3: `1:18048`, `1:19894`, `1:19952`) + **оторванный абсолютный** элемент `1:20452` поверх `Heading` (y≈2349) → секции брать строго по node id (карта секций ниже).
5. **Растровые мокапы** (71 image, 29 mask group, 117 rectangle) → экспортировать ассетами заранее, не верстать «по слоям».

## 4. Дизайн-система — что извлекаем (снизу вверх)

Из переменных Figma в файле есть и ядро ДС, и легаси-токены из кейсов (Inter/Untitled-UI, MW blue, Spherum red и т.п.). **Берём только ядро**, легаси игнорируем.

### 4.1 Токены (`tokens.css`)

Именование — **чистые семантические имена без легаси-префикса `mtk-`** (Webflow здесь нет, мусор не тащим).

Ядро, подтверждённое из `get_variable_defs` (значения сверяются точечно при вёрстке):

- **Цвета:**
  - `--color-ink: #333333` (основной текст / `text/primary`)
  - `--color-text-secondary: #4a4a4a`
  - `--color-text-quaternary: #8a8a8a`
  - `--color-bg: #ffffff` (`background/page`)
  - `--color-sand: #f3efed` (`background/sunken`, плашки)
  - `--color-subtle: #f2f2f2`
  - `--color-brand-red: #f81c1c` (`border/brand`, `color/red/500`)
  - `--color-accent: #fe583e`
  - `--color-cta: #6cd6ab` (зелёная кнопка `button/cta/bg`)
  - `--color-ink-bold: #333333` (тёмные карточки — уточнить, в макете почти чёрный)
  - нейтральная шкала `--color-neutral-{0,100,300,700}`: `#fff / #f3efed / #a3a3a3 / #4a4a4a`
  - бордеры: `--border-default: #bdbdbd`, `--border-strong: #a3a3a3`, `--border-subtle: #f3efed`
- **Типографика (Factor A / Spectral):**
  - Factor A Bold 44/46, Bold 24/30, Bold 22/26, Regular 40/52, Regular 18/22
  - Spectral Regular 22/26 (ls -2), Regular 20/24 (ls -2)
  - оформляем как `--font-display`, `--font-serif` + шкала размеров/line-height
- **Spacing-шкала** (px): 0,4,8,12,16,20,24,28,32,36,40,48,64,80 → `--space-*`
- **Радиусы:** `--radius-2xl: 16`, `--radius-4xl: 32` (+ промежуточные по факту)

> Точные значения каждого токена подтверждаются при вёрстке через Figma MCP, список выше — стартовый каркас.

### 4.2 Примитивы (`components/ui/`)

Выделяем то, что переиспользуется на странице: `Button` (3 варианта: dark, outline, cta-green), `Tag`/`Pill`, `Card`, `Accordion` (для FAQ), `StarRating`, `Avatar`.

### 4.3 Чанк/layout

`Base.astro` — общий каркас страницы. `Nav.astro`, `Footer.astro` — общие для всего сайта (понадобятся и на этапах ②/③).

## 5. Первая страница — секции

Собирается из компонентов `sections/`. 12 блоков (сверху вниз):

1. `Nav` — лого MOTKA + ссылки (Portfolio, About, Design Subscription, Contact us)
2. `Hero` — заголовок, подзаголовок, теги аудитории, 2 CTA, визуальный коллаж
3. `BenefitsCards` — 3 карточки «built for teams…»
4. `Statement` — центрированное утверждение
5. `DesignKinds` — список типов дизайна + тёмная карточка «Keep Your Challenge» + чат-UI + табы категорий
6. `PriorityExamples` — 2 примера со спринт-бейджами и статистикой
7. `ClientLogos` — ряд логотипов
8. `Testimonials` — 3 отзыва + ссылка на Clutch
9. `Process` — интеграции + 3 шага
10. `Pricing` — 3 тарифа + «what happens after you subscribe»
11. `Faq` — аккордеон + тёмная CTA-карточка
12. `Footer` — колонки + лого + копирайт

Вёрстка идёт **секция за секцией** по pixel-perfect воркфлоу (см. §6).

## 6. Pixel-perfect воркфлоу (на каждую секцию)

1. Снять точные числа из Figma (`get_metadata` / `get_design_context` по node секции): размеры, отступы, позиции, font-size/line-height.
2. Сверстать секцию на токенах.
3. Headless-скриншот результата (локальный билд / dev-сервер).
4. Измерить пиксели кодом (PIL): края, размеры, гэпы, выравнивание; сопоставить с Figma.
5. Перечислить расхождения с px-дельтой, починить, повторять до совпадения.
6. Отличать реальные баги (неверный отступ/размер — чинить) от суб-пиксельных различий рендера шрифтов (отметить, не форсить).

## 6а. Адаптив (брейкпойнты)

Мобильного макета **нет** → раскладки на узких брейках — мои дизайн-решения (правила рефлоу), ревью по результату. Pixel-perfect к адаптиву не применяется.

**Брейкпойнты:** `1440` (база, pixel-perfect) → `1280` → `1024` → `768` → `390`.

Подход: десктоп 1440 как основа, вниз — осмысленный рефлоу каждой секции (колонки → стопка, уменьшение типографики/отступов по шкале токенов, hero-лента → адаптированный показ). Проверка на каждом брейке: нет горизонтального скролла, контент читаем, ничего не наезжает. Лейаут секций пересобираем руками (см. риск №1 в §4а), а не полагаемся на авто-лейаут Figma.
